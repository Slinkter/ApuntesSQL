  • 🏠 Índice

📌 Objetivos: Crear índices, analizar planes de ejecución, optimizar consultas.

1. Tipos de Índices

TipoUsoCuándo usarlo
B-treeDefault, búsqueda exacta y rangoWHERE, ORDER BY
HashSolo igualdadWHERE col = valor
GINÍndice invertidoArray, JSON, texto completo
GiSTEspacial, textoGeometría, rangos
BRINRange/minmaxTablas grandes secuenciales

2. Crear Índices

-- Índice B-tree (default)
CREATE INDEX idx_empleados_dept ON empleados(department);

-- Índice único
CREATE UNIQUE INDEX idx_email ON usuarios(email);

-- Índice compuesto
CREATE INDEX idx_pedidos_fecha_cliente ON pedidos(fecha, id_cliente);

-- Índice parcial (solo filas específicas)
CREATE INDEX idx_ventas_2024 ON ventas(fecha) 
WHERE fecha >= '2024-01-01';

-- Índice sobre expresión
CREATE INDEX idx_nombre_upper ON empleados(UPPER(nombre));

-- Índice GIN para JSON
CREATE INDEX idx_datos_json ON tabla USING GIN(datos);

3. EXPLAIN - Analizar Consultas

-- Ver plan de ejecución (sin ejecutar)
EXPLAIN SELECT * FROM empleados WHERE department = 'Ventas';

-- Ejecutar y ver tiempo real
EXPLAIN ANALYZE 
SELECT * FROM empleados WHERE salary > 50000;

-- Ejemplo de output:
-- Index Scan using idx_empleados_dept on empleados
-- (cost=0.00..4.25 rows=1 width=100)
-- (actual time=0.001..0.002 rows=5 loops=1)

Costo (cost): Valor estimado de operaciones. Lower es mejor. Format: inicio..total.

4. Optimización

  • ANALYZE: Actualizar estadísticas para mejores planes
  • CLUSTER: Ordenar tabla según índice
  • VACUUM: Recuperar espacio de filas eliminadas
  • Verificar uso: pg_stat_user_indexes
-- Actualizar estadísticas
ANALYZE empleados;

-- Reconstruir índice
REINDEX INDEX idx_empleados_dept;

-- Vacuum (recuperar espacio)
VACUUM ANALYZE empleados;

📝 Resumen: Los índices aceleran búsquedas. B-tree es el más común. EXPLAIN ANALYZE muestra el plan real de ejecución. ANALYZE actualiza estadísticas para mejores decisiones del planner.
