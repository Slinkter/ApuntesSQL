📌 Objetivos: Comprender transacciones, isolation levels, y manejo de concurrencia.

1. Transacciones

-- Transacción básica
BEGIN;
    UPDATE cuentas SET saldo = saldo - 100 WHERE id = 1;
    UPDATE cuentas SET saldo = saldo + 100 WHERE id = 2;
COMMIT;

-- Deshacer transacción
BEGIN;
    INSERT INTO pedidos (...) VALUES (...);
ROLLBACK;

-- Savepoint
BEGIN;
    INSERT INTO tabla1 VALUES (1);
    SAVEPOINT sp1;
    INSERT INTO tabla1 VALUES (2);
    ROLLBACK TO SAVEPOINT sp1;  -- Solo deshace el 2do insert
COMMIT;

2. Isolation Levels

LevelDescripciónProblema
READ UNCOMMITTEDLee datos sin confirmarDirty Read
READ COMMITTEDSolo lee datos confirmadosNon-repeatable Read
REPEATABLE READResultados consistentesPhantom Read
SERIALIZABLEEjecución secuencialMás lento
-- Setear isolation level
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;

-- En función
ALTER FUNCTION mi_funcion() SET isolation_level = SERIALIZABLE;

3. MVCC en PostgreSQL

Concepto: MVCC (Multi-Version Concurrency Control) permite que múltiples transacciones trabajen simultáneamente sin bloqueos. Cada transacción ve una "snapshot" de los datos.

4. Locking

-- Lock explícito
LOCK TABLE empleados IN EXCLUSIVE MODE;

-- Lock con timeout
LOCK TABLE empleados IN ACCESS EXCLUSIVE MODE NOWAIT;

-- Advisory locks
SELECT pg_advisory_lock(12345);
-- hacer trabajo --
SELECT pg_advisory_unlock(12345);

📝 Resumen: Las transacciones agrupan operaciones atómicamente. Los isolation levels controlan la visibilidad de datos no confirmados. MVCC permite concurrencia sin bloqueos. Los locks sincronizan acceso a recursos.

EJERCICIO

Ejecutar transferencia de 100 entre cuentas 1 y 2 con rollback si falla.

BEGIN;
    UPDATE cuentas SET saldo = saldo - 100 WHERE id = 1;
    UPDATE cuentas SET saldo = saldo + 100 WHERE id = 2;
    -- Verificar
    IF (SELECT saldo FROM cuentas WHERE id = 1) < 0 THEN
        ROLLBACK;
    ELSE
        COMMIT;
    END IF;
