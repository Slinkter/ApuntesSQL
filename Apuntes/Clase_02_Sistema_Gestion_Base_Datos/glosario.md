# Glosario - Clase 02: Modelo Relacional

*   **ACID (Atomicity, Consistency, Isolation, Durability):** Propiedades fundamentales que garantizan la fiabilidad de las transacciones de base de datos.
*   **Arquitectura de Tres Niveles (ANSI/SPARC):** Un modelo que divide la base de datos en tres esquemas: Interno, Conceptual y Externo, para proporcionar independencia de datos.
*   **Atributo:** En el modelo relacional, es una característica o propiedad de una entidad, representada como una columna en una tabla.
*   **Catálogo del Sistema (Metadatos):** Un diccionario de datos que almacena información sobre la estructura de la base de datos, objetos, usuarios y permisos.
*   **Control de Acceso (SGBD):** Mecanismos para asegurar que solo usuarios autorizados accedan y operen sobre los datos.
*   **Control de Concurrencia:** Mecanismo del SGBD que permite a múltiples usuarios acceder y modificar la base de datos simultáneamente sin interferir entre sí.
*   **DBMS (Database Management System):** Software que gestiona, define, crea, mantiene y controla el acceso a una base de datos.
*   **Esquema Conceptual:** La visión global de la base de datos para una comunidad de usuarios, incluyendo todos los datos, interrelaciones y restricciones.
*   **Esquema Externo (Vista):** La visión personalizada de la base de datos que ve un usuario o aplicación específica, adaptada a sus necesidades.
*   **Esquema Interno:** Describe la estructura física de almacenamiento de la base de datos en los dispositivos de hardware.
*   **Foreign Key (FK):** Clave Foránea. Un atributo (o conjunto de atributos) en una tabla que se refiere a la Primary Key de otra tabla, estableciendo una relación entre ellas.
*   **Independencia de Datos:** Capacidad de modificar el esquema de un nivel de la base de datos sin afectar el esquema del nivel superior.
*   **Independencia Física de Datos:** Capacidad de cambiar la estructura física de almacenamiento sin afectar el esquema conceptual o externo.
*   **Independencia Lógica de Datos:** Capacidad de cambiar el esquema conceptual sin afectar los esquemas externos o las aplicaciones de usuario.
*   **Integridad de Datos (SGBD):** La capacidad del SGBD para mantener la precisión, consistencia y fiabilidad de los datos.
*   **Lenguaje de Definición de Datos (DDL):** Comandos para crear y modificar la estructura de la base de datos (ej. `CREATE`, `ALTER`, `DROP`).
*   **Lenguaje de Manipulación de Datos (DML):** Comandos para insertar, recuperar, modificar y eliminar datos (ej. `SELECT`, `INSERT`, `UPDATE`, `DELETE`).
*   **Metadatos:** "Datos sobre los datos", información que describe la estructura y propiedades de los datos.
*   **Motor de Base de Datos:** Componente central del SGBD responsable del almacenamiento, recuperación y actualización física de los datos.
*   **Primary Key (PK):** Clave Primaria. Un atributo (o conjunto de atributos) que identifica de forma única cada tupla (fila) en una relación (tabla). Debe ser única y no nula.
*   **Recuperación y Respaldo (SGBD):** Funcionalidades para crear copias de seguridad y restaurar la base de datos después de fallos.
*   **Relación:** En el modelo relacional, sinónimo de tabla, una colección de tuplas (filas) y atributos (columnas).
*   **SGBD (Sistema de Gestión de Base de Datos):** Software que permite definir, crear, mantener y controlar el acceso a la base de datos.
*   **Transacción:** Una operación lógica completa de lectura o escritura sobre la base de datos.
*   **Tupla:** En el modelo relacional, sinónimo de fila o registro en una tabla.
