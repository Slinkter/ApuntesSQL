# Glosario - Clase 06: Modelamiento de Datos en la Empresa

*   **Modelamiento de Datos:** El proceso de crear una representación abstracta y estructurada de los datos de un sistema, incluyendo sus propiedades, relaciones y restricciones.
*   **Modelo Conceptual de Datos:** Una representación de alto nivel de los datos, independiente de la tecnología, que describe las entidades de interés, sus atributos y las relaciones entre ellas desde la perspectiva del usuario.
*   **Diagrama de Entidad-Relación (DER):** Una notación gráfica común para representar un modelo conceptual de datos, mostrando entidades como rectángulos, atributos como óvalos y relaciones como rombos.
*   **Entidad (Modelamiento):** Un objeto o concepto del mundo real (persona, lugar, cosa, evento) sobre el cual se desea almacenar información en la base de datos.
*   **Atributo (Modelamiento):** Una característica o propiedad de una entidad que describe un dato específico de esa entidad.
*   **Relación (Modelamiento):** Una asociación o interacción significativa entre dos o más entidades.
*   **Cardinalidad (Modelamiento):** En un DER, especifica el número de instancias de una entidad que pueden asociarse con el número de instancias de otra entidad en una relación (ej. uno a uno, uno a muchos, muchos a muchos).
*   **Opcionalidad:** Indica si la participación de una entidad en una relación es obligatoria (debe existir) u opcional (puede no existir).
*   **Modelo Lógico de Datos:** Una representación detallada de los datos que define la estructura del esquema de la base de datos, siguiendo las reglas de un modelo de datos específico (ej. relacional: tablas, columnas, claves primarias, claves foráneas).
*   **Tabla (Modelo Lógico):** La representación de una entidad en el modelo relacional lógico, compuesta por filas y columnas.
*   **Columna (Modelo Lógico):** La representación de un atributo en el modelo relacional lógico, con un nombre y un tipo de dato.
*   **Clave Primaria (Modelo Lógico):** Un atributo o conjunto de atributos que identifica de forma única cada fila en una tabla del modelo lógico.
*   **Clave Foránea (Modelo Lógico):** Un atributo o conjunto de atributos en una tabla que hace referencia a la clave primaria de otra tabla, estableciendo una relación entre ellas.
*   **Modelo Físico de Datos:** La implementación real del modelo lógico de datos en un SGBD específico, incluyendo detalles como tipos de datos específicos del proveedor, índices, vistas, particiones y almacenamiento.
*   **Herramientas CASE (Computer-Aided Software Engineering):** Aplicaciones de software que automatizan o asisten en las tareas de diseño, desarrollo y mantenimiento de sistemas de información, incluyendo el modelamiento de datos.
*   **Normalización (Modelamiento):** Un proceso sistemático para evaluar y refinar el diseño de un esquema de base de datos relacional para reducir la redundancia y mejorar la integridad de los datos.
*   **Diseño de Bases de Datos:** El proceso global de crear la estructura de una base de datos para satisfacer las necesidades de información de los usuarios y las aplicaciones.
