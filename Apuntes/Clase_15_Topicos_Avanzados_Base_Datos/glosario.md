# Glosario - Clase 15: Data Warehouse (Arquitectura y Explotación)

*   **Análisis OLAP (Online Analytical Processing):** Un enfoque para analizar datos multidimensionales que permite a los usuarios realizar consultas complejas, ver los datos desde diferentes perspectivas y realizar agregaciones rápidamente.
*   **Arquitectura de Data Warehouse:** El diseño estructurado de un Data Warehouse, que incluye las fuentes de datos, el área de staging, el motor ETL, el propio DW y las herramientas de acceso/análisis.
*   **Business Intelligence (BI):** Conjunto de productos y estrategias que ayudan a los usuarios a acceder y analizar rápidamente la información para la toma de decisiones estratégicas.
*   **Cubos de Datos:** Estructura de datos multidimensional utilizada en OLAP para almacenar y organizar grandes volúmenes de datos preagregados, permitiendo un análisis rápido desde múltiples perspectivas.
*   **Data Mart:** Un subconjunto de un Data Warehouse, más pequeño y enfocado a un área de negocio o departamento específico (ej. ventas, finanzas), diseñado para satisfacer necesidades analíticas particulares.
*   **Data Mining (Minería de Datos):** Proceso de descubrir patrones, tendencias y conocimientos significativos en grandes conjuntos de datos, utilizando técnicas estadísticas y de inteligencia artificial.
*   **Data Mining de Descubrimiento:** Tipo de minería de datos que busca patrones o relaciones ocultas en los datos sin hipótesis previas.
*   **Data Mining Predictivo:** Tipo de minería de datos que utiliza datos históricos para construir modelos que pronostican valores futuros o comportamientos.
*   **Data Warehouse (DW):** Un repositorio centralizado de datos, optimizado para el análisis (no para transacciones diarias), que almacena información histórica, integrada, organizada por temas y no volátil.
*   **DW Integrado:** Característica de un DW que significa que los datos de múltiples fuentes heterogéneas se consolidan y se hacen consistentes.
*   **DW No Volátil:** Característica de un DW que indica que los datos, una vez cargados, no se modifican ni se eliminan; los datos históricos se mantienen.
*   **DW Organizado por Temas:** Característica de un DW que agrupa los datos en torno a los principales sujetos u objetos de negocio (ej. clientes, productos, ventas), en lugar de aplicaciones transaccionales.
*   **DW Variante en el Tiempo:** Característica de un DW que significa que los datos contienen un componente histórico y pueden ser analizados para observar tendencias y cambios a lo largo del tiempo.
*   **ETL (Extracción, Transformación y Carga):** El proceso fundamental en la construcción de un Data Warehouse que implica extraer datos de diversas fuentes, transformarlos (limpiar, validar, consolidar) y cargarlos en el DW.
*   **Explotación de Datos:** El proceso de utilizar las herramientas y técnicas adecuadas (ej. OLAP, Data Mining, reporting) para extraer valor y conocimiento de los datos almacenados en un Data Warehouse.
*   **Metadatos (en DW):** Datos sobre los datos en un Data Warehouse, que describen su origen, su estructura, las transformaciones aplicadas, las reglas de negocio y los usuarios, esenciales para entender el contexto de la información.
*   **OLAP (Online Analytical Processing):** Tecnología para el análisis multidimensional de datos que permite una visualización y agregación rápida de grandes volúmenes de datos.
*   **OLTP (Online Transaction Processing):** Sistemas transaccionales de procesamiento en línea, diseñados para el manejo eficiente y rápido de transacciones operativas diarias (ej. ingreso de pedidos, transacciones bancarias).
*   **Query Ad hoc:** Consultas específicas y no predefinidas que los usuarios finales pueden formular sobre la marcha para explorar los datos y responder preguntas puntuales.
