1. Comandos DDL en PostgreSQL

  • CREATE: Crea objetos (tablas, índices, secuencias)
  • ALTER: Modifica estructura de objetos existentes
  • DROP: Elimina objetos de forma permanente
  • TRUNCATE: Elimina todas las filas (más rápido que DELETE)

3. Constraints (Restricciones)

Concepto clave: Los constraints son reglas que garantizan la integridad de los datos. Se definen al crear la tabla o después con ALTER TABLE.

ConstraintDescripciónEjemplo
PRIMARY KEYIdentificador único, no nuloid SERIAL PRIMARY KEY
FOREIGN KEYReferencia a otra tablaREFERENCES clientes(id)
NOT NULLEl campo es obligatorionombre VARCHAR NOT NULL
UNIQUEValores no repetidosemail VARCHAR UNIQUE
CHECKCondición personalizadaCHECK (precio > 0)
DEFAULTValor por defectoestado VARCHAR DEFAULT 'A'

4. Ejemplos Prácticos

-- Crear tabla con constraints
CREATE TABLE productos (
    id          SERIAL PRIMARY KEY,
    nombre      VARCHAR(100) NOT NULL UNIQUE,
    descripcion VARCHAR(500),
    precio      NUMERIC(10,2) NOT NULL CHECK (precio >= 0),
    stock       INTEGER DEFAULT 0 NOT NULL,
    categoria   VARCHAR(50)
);
-- Tabla con Foreign Key y cascade
CREATE TABLE pedidos (
    id_pedido   SERIAL PRIMARY KEY,
    id_cliente  INTEGER NOT NULL,
    total       NUMERIC(12,2),
    fecha       DATE DEFAULT CURRENT_DATE,
    CONSTRAINT fk_cliente 
        FOREIGN KEY (id_cliente) 
        REFERENCES clientes(id_cliente)
        ON DELETE CASCADE
);

5. ALTER TABLE

-- Añadir columna
ALTER TABLE productos ADD COLUMN fecha_creacion TIMESTAMP;

-- Modificar columna
ALTER TABLE productos ALTER COLUMN nombre TYPE VARCHAR(150);

-- Añadir constraint
ALTER TABLE productos ADD CONSTRAINT precio_positivo CHECK (precio > 0);

-- Eliminar columna
ALTER TABLE productos DROP COLUMN descripcion;

-- Eliminar constraint
ALTER TABLE productos DROP CONSTRAINT precio_positivo;

6. Integridad Referencial

AcciónDescripción
ON DELETE CASCADEElimina hijos automáticamente
ON DELETE SET NULLPone la FK en NULL
ON DELETE RESTRICTImpide eliminar si hay hijos
ON DELETE NO ACTIONNo hace nada (genera error)

📝 Resumen: DDL permite definir la estructura de la base de datos. PostgreSQL usa SERIAL para auto-incremento, VARCHAR para texto, NUMERIC para decimales. Los constraints como PRIMARY KEY, FOREIGN KEY y CHECK garantizan la integridad de los datos.

EJERCICIO RESUELTO

Enunciado:

Crear esquema para librería: AUTORES (ID, Nombre), LIBROS (ISBN PK, Titulo, ID_Autor FK, Precio), CLIENTES (ID PK, Email UNIQUE), PRESTAMOS (ID, ID_Cliente FK, ISBN FK).

Solución:

CREATE TABLE autores (
    id_autor   SERIAL PRIMARY KEY,
    nombre     VARCHAR(100) NOT NULL
);

CREATE TABLE libros (
    isbn     VARCHAR(13) PRIMARY KEY,
    titulo   VARCHAR(255) NOT NULL,
    id_autor INTEGER NOT NULL REFERENCES autores(id_autor),
    precio   NUMERIC(7,2) CHECK (precio > 0)
);

CREATE TABLE clientes (
    id_cliente SERIAL PRIMARY KEY,
    email      VARCHAR(255) UNIQUE NOT NULL
);

CREATE TABLE prestamos (
    id_prestamo SERIAL PRIMARY KEY,
    id_cliente  INTEGER REFERENCES clientes(id_cliente),
    isbn        VARCHAR(13) REFERENCES libros(isbn),
    fecha       DATE DEFAULT CURRENT_DATE
);
