1. ¿Qué es un DBMS?

Definición

Software que actúa como intermediario entre usuarios y la base de datos. Permite definir, crear y controlar el acceso a los datos.

Función del DBMS: Abstraer la complejidad del almacenamiento físico y proporcionar herramientas para gestionar datos de manera eficiente y segura.

2. Arquitectura de Tres Niveles

  • Nivel Interno (Físico): Cómo se almacenan los datos en disco
  • Nivel Conceptual (Lógico): Qué datos existen y sus relaciones - visión global
  • Nivel Externo (Vistas): Cómo cada usuario ve la información

Analogía: Es como un edificio: el plano arquitectónico (interno), la descripción formal a constructores (conceptual), y los diferentes departamentos con sus propias vistas del edificio (externo).

3. Funciones del DBMS

  • Gestión de Usuarios: Permisos y control de acceso
  • Gestión de Rendimiento: Optimización de consultas
  • Backup y Recovery: Respaldo y restauración
  • Gestión de Transacciones: Propiedades ACID

4. Propiedades ACID

PropiedadDescripciónEjemplo
AtomicidadTodo o nadaTransferencia bancaria
ConsistenciaEstado válido siempreSaldo no negativo
AislamientoTransacciones independientesConcurrentes no se interfieren
DurabilidadCambios permanentes即使 sistema falla

5. Lenguajes SQL

  • DDL (Data Definition): CREATE, ALTER, DROP - estructura
  • DML (Data Manipulation): SELECT, INSERT, UPDATE, DELETE - datos
  • DCL (Data Control): GRANT, REVOKE - permisos
  • TCL (Transaction Control): COMMIT, ROLLBACK - transacciones

6. Componentes del DBMS

  • Procesador de Consultas: Interpreta y optimiza SQL
  • Gestor de Transacciones: Asegura ACID
  • Motor de BD: Almacena y recupera datos
  • Catálogo del Sistema: Metadatos (esquemas, tipos, índices)

EJERCICIO RESUELTO

Enunciado:

Un banco transfiere 100 USD de cuenta A a cuenta B. Explica cómo cada propiedad ACID se aplica.

Atomicidad: La operación completa (restar A + sumar B) se ejecuta o no se ejecuta en absoluto.
Consistencia: La suma total (A+B) debe ser igual antes y después.
Aislamiento: Otros usuarios ven estado anterior o posterior, nunca intermedio.
Durabilidad: Una vez commit, los cambios son permanentes.

QUIZ

P1: ¿Qué comando pertenece a DDL?

R1: CREATE TABLE (define estructura)

P2: ¿Qué propiedad ACID garantiza que una transacción se ejecute completa o no se ejecute?

R2: Atomicidad

P3: GRANT es parte de qué lenguaje?

R3: DCL (Data Control Language)
