# Glosario - Clase 09: Otras Formas Normales y Desnormalización

*   **4FN (Cuarta Forma Normal):** Una relación está en 4FN si está en FNBC y no contiene dependencias multivaluadas no triviales. ¡Es un paso más allá para que tus datos no tengan "doble personalidad" sin razón!
*   **Anomalías (Normalización):** Problemas de redundancia e inconsistencia que surgen en una base de datos mal diseñada, como la imposibilidad de insertar datos sin inventar otros (inserción), la pérdida de información al borrar (eliminación) o la necesidad de actualizar muchos lugares a la vez (modificación).
*   **Dependencia de Unión:** Una propiedad de una tabla que permite que se reconstruya a partir de la unión (JOIN) de dos o más de sus proyecciones. Es la base de la 5FN.
*   **Dependencia Funcional:** Una relación entre atributos donde el valor de uno (o un grupo) determina el valor de otro. Si A determina B, escribimos A → B.
*   **Dependencia Multivaluada:** Ocurre si un atributo (o grupo de atributos) determina un conjunto de valores para otro atributo, y ese conjunto de valores es independiente de los demás atributos no clave. ¡Es como tener opciones independientes en una misma "fila"!
*   **Desnormalización:** El acto de introducir intencionadamente redundancia en una base de datos. ¡Es como romper un poco las reglas de la normalización, pero con un buen motivo: hacer las consultas más rápidas!
*   **FNBC (Forma Normal de Boyce-Codd):** Una forma normal más estricta que la 3FN. Una relación está en FNBC si cada determinante (atributo que determina a otro) es también una clave candidata. Ayuda a resolver problemas donde la 3FN se queda corta.
*   **Normalización:** El proceso de limpiar y mejorar el diseño lógico de una base de datos para reducir la redundancia y mejorar la integridad. ¡Es como organizar tu armario para que todo tenga su lugar y no tengas diez camisetas iguales escondidas!
*   **Redundancia Intencional:** Duplicación de datos que se introduce deliberadamente durante la desnormalización para mejorar el rendimiento de lectura.
*   **5FN (Quinta Forma Normal):** Una relación está en 5FN si está en 4FN y no contiene dependencias de unión no triviales, lo que significa que no puede descomponerse en tablas más pequeñas sin perder información.
