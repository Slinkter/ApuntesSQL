1. Creación de Tablas Relacionales (Foreign Keys)

[Continuación temporal... ~01:15:00]

El profesor retoma la creación de la última tabla esencial para un E-commerce: orders (compras). El objetivo es entender cómo se vinculan los "documentos únicos" (IDs) de las distintas entidades en una Base de Datos Relacional.

El profesor utiliza la herramienta Miro para dibujar flechas entre la tabla de Usuarios, Productos y Órdenes. Explica que la llave foránea (Foreign Key) es lo que permite que una compra sepa exactamente quién es el usuario que la efectuó y qué producto llevó, asegurando que no existan compras huérfanas en el sistema.

Una FOREIGN KEYClave Foránea: Vincula una tabla con el identificador de otra es una restricción que asegura que el valor introducido en una columna debe existir obligatoriamente en la llave primaria (PRIMARY KEYClave Primaria: Identificador único e irrepetible (el DNI de la fila)) de otra tabla.

Código Completo de la Tabla Relacional: 
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEYClave Primaria: Identificador único e irrepetible (el DNI de la fila),
    user_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT DEFAULT 1,
    total DECIMAL(10,2),
    order_date DATE DEFAULT CURRENT_DATE,
    FOREIGN KEYClave Foránea: Vincula una tabla con el identificador de otra (user_id) REFERENCES users(id),
    FOREIGN KEYClave Foránea: Vincula una tabla con el identificador de otra (product_id) REFERENCES products(id)
);
El profesor mencionó que el campo total podría ser una "columna calculada" (total DECIMAL(10,2) AS (price * quantity)), pero decide dejarlo como un campo normal por ahora para no complicar la sintaxis introductoria. Es fundamental que los campos user_id y product_id compartan exactamente el mismo tipo de dato que sus llaves primarias de origen (en este caso INT).

2. DMLData Manipulation Language: Manipula registros vivos (INSERT, UPDATE, DELETE): Inserción de Registros (INSERT)

[~01:25:00]

El profesor transiciona al DMLData Manipulation Language: Manipula registros vivos (INSERT, UPDATE, DELETE) (Data Manipulation Language). Mientras el DDLData Definition Language: Define la estructura inmutable (CREATE, DROP, ALTER) define la estructura, el DMLData Manipulation Language: Manipula registros vivos (INSERT, UPDATE, DELETE) manipula los datos vivos que habitan esas estructuras.

Sintaxis de Inserción Básica

INSERT INTO users (name, email, password, cellphone)
VALUES 
('Sergie Code', 'info@sergiecode.com', 'secreto123', '+4412345'),
('Ricardo Darin', 'ricardo@darin.com', 'secreto456', '+1198765'),
('Jen Ortega', 'jen@ortega.com', 'merlina123', '+11223344');

Inserción Parcial y Comportamiento de Nulos/Defaults

Cuando se inserta un registro y se omiten campos, si el campo permite nulos se llena con NULL, y si tiene un constraint DEFAULT, toma ese valor. Si se exige explícitamente pasar por todos los valores (sin declarar las columnas arriba), se debe pasar la palabra clave NULL manualmente.

Ejemplo del profesor: 
INSERT INTO products (name, description, price, stock)
VALUES ('Notebook', 'Laptop alta gama', 1000.50, 10);

-- Qué pasa si omitimos columnas permitidas a ser Nulas:
INSERT INTO products (name, price)
VALUES ('Mouse', 5.50);
-- El motor insertará NULL en description y 0 en stock (porque tenía DEFAULT 0).

3. DMLData Manipulation Language: Manipula registros vivos (INSERT, UPDATE, DELETE): Edición y Borrado Crítico (UPDATE, DELETE, TRUNCATE)

[~01:40:00]

UPDATE (Actualización)

Sirve para modificar datos existentes de una o múltiples columnas a la vez separándolas por comas.

📌 ÉNFASIS DEL PROFESOR (EL PELIGRO DEL WHERE):
"¡JAMÁS hagas un UPDATE sin la cláusula WHERE! Si olvidas el WHERE, modificarás todas las filas. Por ejemplo, le cambiarías la contraseña a TODOS los usuarios del sistema por accidente. Es el meme típico de SQL por una razón real." 
-- Esto cambia TODOS los registros de la tabla
UPDATE users SET is_active = false; 

-- Consola: 105,432 rows affected. (¡Acabas de destruir la BD productiva!)

Las plataformas ORM de alto nivel suelen tener un "Safe Mode" que lanza una excepción si intentas ejecutar un UPDATE sin un condicional.

UPDATE users 
SET password = 'the_thing_is_my_friend', cellphone = '+150011222'
WHERE id = 3;

DELETE, DROP y TRUNCATE

El profesor establece una diferencia crucial entre las tres formas de eliminar información:

  • DELETE FROM tabla WHERE condicion; → Borra filas específicas. Si omites el WHERE, borra todas las filas pero la estructura de la tabla y sus contadores internos se mantienen.
  • TRUNCATE TABLE tabla; → Vacía la tabla por completo y reinicia el contador del AUTO_INCREMENT a 1. La tabla sigue existiendo.
  • DROP TABLE tabla; → DDLData Definition Language: Define la estructura inmutable (CREATE, DROP, ALTER) puro. Destruye la tabla por completo. Deja de existir la estructura en la base de datos.
Regla de Integridad de IDs Autoincrementales: El profesor hizo un experimento borrando el producto ID 5. Al crear uno nuevo, el motor generó el ID 8 (continuando la secuencia). Las bases de datos relacionales NUNCA reciclan IDs de registros borrados. Si borras el 5, el 5 se pierde para siempre. Esto garantiza la integridad histórica.

Bloqueo por Foreign Key: El profesor intentó borrar todos los productos y falló porque algunos productos ya estaban "comprados" en la tabla orders. La Base de Datos rechazó la eliminación para evitar dejar órdenes de compra con productos "fantasmas". 

DELETE FROM products WHERE id = 5;
-- ERROR: update or delete on table "products" violates foreign key constraint 
-- "fk_product_id" on table "orders".
-- DETAIL: Key (id)=(5) is still referenced from table "orders".

4. DQLData Query Language: Extrae información sin alterarla (SELECT): Consultas de Extracción de Datos (SELECT)

[~01:50:00]

DQLData Query Language: Extrae información sin alterarla (SELECT) (Data Query Language). Consiste primordialmente en el comando SELECT para extraer y consultar información.

Operadores y Filtros Múltiples

Se pueden concatenar condiciones complejas usando AND / OR.

SELECT name, price 
FROM products 
WHERE stock >= 10 AND price > 10;

Comodines de Texto (LIKE)

Permite buscar patrones dentro de un texto. El porcentaje (%) reemplaza a "cualquier cadena de caracteres".

SELECT name FROM products WHERE name LIKE '%mo%';

Nota del profesor: LIKE en MySQL no distingue entre mayúsculas y minúsculas (case insensitive) ni es sensible a acentos (automóvil vs automovil) bajo la configuración por defecto de UTF-8.

IN, BETWEEN y Valores Nulos

  • Conjuntos: WHERE id IN (1, 2, 3) evalúa si el ID pertenece a esa lista.
  • Rangos: WHERE price BETWEEN 50 AND 150 incluye los bordes 50 y 150.
  • Nulidad: WHERE description IS NULL (no se usa = NULL).

Orden, Limites y Duplicados

  • ORDER BY: Organiza la salida. ORDER BY price DESC (mayor a menor) o ASC (menor a mayor).
  • LIMIT: Restringe la cantidad de datos devueltos. LIMIT 3 (solo trae los primeros 3 registros). Vital para paginación de software.
  • DISTINCT: Filtra valores repetidos proyectados. SELECT DISTINCT name FROM products;
  • AS (Alias): Renombra la salida para el backend temporalmente. SELECT name AS producto FROM products;

5. DQLData Query Language: Extrae información sin alterarla (SELECT): Introducción a Funciones de Agregación

[~02:00:00]

Las funciones de agregación devuelven un único resultado tras evaluar múltiples filas.

  • COUNT(*): Cuenta la cantidad de filas resultantes.
  • SUM(columna): Suma todos los valores numéricos de esa columna. Ejemplo: Ganancia total del día.
  • AVG(columna): Average (Promedio). Ejemplo: Precio promedio de venta para segmentación de mercado.
SELECT SUM(total) AS ganancia_total FROM orders;
SELECT COUNT(*) AS total_productos_stock FROM products WHERE stock >= 1;

GLOSARIO TÉCNICO

Foreign Key (Clave Foránea):
Una columna en una tabla que actúa como un puntero a la PRIMARY KEYClave Primaria: Identificador único e irrepetible (el DNI de la fila) de otra tabla, forzando la integridad de las relaciones.

DMLData Manipulation Language: Manipula registros vivos (INSERT, UPDATE, DELETE) (Data Manipulation Language):
Subconjunto de SQL que manipula la información (CRUD operativo). Comandos: INSERT, UPDATE, DELETE.

DQLData Query Language: Extrae información sin alterarla (SELECT) (Data Query Language):
Lenguaje exclusivo para la consulta de información sin alterarla. Comando base: SELECT.

TRUNCATE vs DROP:
TRUNCATE vacía los datos pero deja la estructura de la tabla viva. DROP aniquila tanto los datos como la estructura misma de la base de datos.

Wildcard (Comodín %):
Símbolo utilizado con la instrucción LIKE que le indica al motor SQL que no importa cuántos caracteres existan en ese espacio.

PREGUNTAS DE REPASO (Estilo Examen)

NIVEL 1 — Comprensión directa

P1: ¿Qué sucede con los números Autoincrementales cuando se elimina un registro mediante la instrucción DELETE?

R1: El número autoincremental no se recicla. Si borras el ID 5, el próximo insert generará el ID 6 (o el siguiente disponible). La secuencia avanza para garantizar que nunca haya choques históricos con datos dependientes.

P2: ¿Por qué falló el intento del profesor de borrar todos los datos de la tabla "products"?

R2: Por la protección de la Foreign Key. Dado que existían órdenes en la tabla "orders" que hacían referencia a los IDs de "products", la base de datos abortó la eliminación para prevenir una violación de integridad referencial.

NIVEL 2 — Aplicación

P3: Requieres traer de la tabla usuarios aquellos cuyo nombre inicie con "Ser" ignorando cómo terminan. ¿Qué query ejecutarías?

R3: Utilizaría el operador LIKE con el comodín al final: SELECT * FROM users WHERE name LIKE 'Ser%';

NIVEL 3 — Análisis / Síntesis

P4: Analiza las diferencias arquitectónicas entre vaciar una tabla usando DELETE (sin WHERE) y usar TRUNCATE. ¿Cuál elegirías para reiniciar un entorno de pruebas?

R4: DELETE evalúa fila por fila (siendo más lento y llenando logs de transacciones) y no reinicia el contador de auto_increment. TRUNCATE elimina la data de golpe a nivel de almacenamiento físico y formatea el contador a 1. Para reiniciar un entorno de pruebas limpiamente, TRUNCATE es la herramienta técnica adecuada.

EJERCICIOS PRÁCTICOS (Basados en el Repositorio Oficial)

Ejercicio 1 — DMLData Manipulation Language: Manipula registros vivos (INSERT, UPDATE, DELETE) (Actualización Crítica)
Objetivo: Identificar usuarios y actualizar precios en masa basándonos en condicionales.
Enunciado: A causa de la inflación, el negocio ha decidido aumentar el precio de todos sus productos en $10.00, pero únicamente de los productos que actualmente cuestan menos de $100.00. Escribe el DMLData Manipulation Language: Manipula registros vivos (INSERT, UPDATE, DELETE) correspondiente.
Solución guiada:

UPDATE products 
SET price = price + 10.00 
WHERE price < 100.00;

Nota: SQL permite usar la misma columna en la fórmula matemática del lado derecho del SET.

Ejercicio 2 — Evolución de Esquemas (ALTER TABLE)
Objetivo: Modificar la estructura de una tabla existente basándonos en la guía oficial del repositorio del profesor.
Enunciado: Se requiere que la tabla users cambie de nombre a customers. Además, se le debe agregar una columna phone de tipo VARCHAR(20) y luego renombrarla a cellphone con un límite de 30 caracteres.
Solución guiada:

-- 1. Renombrar la tabla
ALTER TABLE users RENAME TO customers;

-- 2. Agregar nueva columna
ALTER TABLE customers ADD COLUMN phone VARCHAR(20);

-- 3. Renombrar la columna
ALTER TABLE customers RENAME COLUMN phone TO cellphone;

-- 4. Modificar el tipo y límite de la columna
ALTER TABLE customers MODIFY COLUMN cellphone VARCHAR(30) NOT NULL;
