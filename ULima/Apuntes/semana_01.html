Insight del Arquitecto: El Valor del Dato

Los datos son un activo estrategico: En 2013 las empresas comenzaban a entender que los datos son un recurso competitivo. C.J. Date nos recuerda que una base de datos no es solo un lugar donde guardar cosas — es la representacion formal de la realidad del negocio.

La clave: Datos → Informacion → Conocimiento → Decision. Sin el primer paso, no hay nada.

1. Del Dato al Conocimiento

Conceptos Fundamentales

Los negocios generan datos permanentemente. El dato es un recurso organizacional para competir. Existe necesidad de informacion en todos los niveles de la organizacion.

  • Dato: Representacion simbolica de un hecho. Objetivo, sin contexto. Ej: 3.50
  • Informacion: Dato procesado con contexto. Reduce incertidumbre. Ej: "El precio del producto es 3.50"
  • Conocimiento: Informacion analizada para tomar decisiones. Ej: "El precio subio 75% en 3 anos, debemos ajustar estrategia"

Concepto clave: Los datos son el insumo basico. La informacion surge al procesarlos. El conocimiento emerge al analizar la informacion para la toma de decisiones.

2. El Problema: Sistema Tradicional de Archivos

Antes del enfoque de BD

Cada aplicacion tenia sus propios archivos, generando los siguientes problemas:

Dependencia programa/datos: Cada sistema debe actualizarse con sus propios datos, incluir codigo para la metadata de cada archivo y tener sus propias rutinas de CRUD. Esto produce debil coordinacion y control centralizado.

  • Redundancia de datos: Duplicacion de la misma informacion en multiples archivos. Espacio desperdiciado y mantenimiento costoso.
  • Inconsistencia: Cuando los datos cambian en un archivo pero no en otro, se compromete la integridad de los datos.
  • Limitaciones para compartir datos: No hay forma estandar de acceso entre distintas aplicaciones.
  • Mayor tiempo de desarrollo: Cada sistema reinventa las rutinas de acceso a datos.
  • Sin estandares: No hay coordinacion ni control centralizado sobre los datos.

3. La Solucion: Enfoque de Base de Datos

Base de Datos: Coleccion de datos almacenados en un formato estandar, disenado para que pueda ser compartido por multiples usuarios.

Analogia: El DBMS actua como un bibliotecario experto que conoce la ubicacion exacta de cada libro (dato) y puede recuperarlo rapidamente sin que el usuario necesite saber donde esta fisicamente almacenado.

Ventajas del enfoque de Base de Datos:

  • Independencia programa/datos: Los metadatos se almacenan en el DBMS. Las aplicaciones no necesitan preocuparse por formatos de datos. Las consultas y actualizaciones son gestionadas por el DBMS.
  • Redundancia minima: Se reduce la duplicacion, mejorando consistencia e integridad.
  • Mejor comparticion de datos: Diferentes usuarios obtienen diferentes vistas de los datos.
  • Reforzamiento de estandares: Todos los accesos a datos se realizan de la misma manera.
  • Mejor calidad de datos: Restricciones, reglas de validacion.
  • Accesibilidad: Uso de un lenguaje estandar de consultas (SQL).
  • Seguridad, Backup/Recovery, Concurrencia: La recuperacion ante desastres es mas sencilla.

4. Costos y Riesgos

Tipo Ejemplos
Costos iniciales Software, hardware, comunicaciones, conversion de datos, estudios, diseno, capacitacion
Costos permanentes Personal especializado (DBA), upgrades de software, mantenimiento, soporte
Riesgos Complejidad de instalacion, conflictos organizacionales, determinacion de procesos de backup

5. Componentes del Entorno de BD

Componente Funcion
Herramientas CASE Herramientas usadas para disenar BD y aplicaciones
Aplicacion Programa usado para crear, modificar y leer en la BD
Repositorio Base de conocimientos donde se guardan definiciones de datos, relaciones y formatos
Base de Datos Coleccion organizada de datos logicamente relacionados
DBMS Software empleado para crear, mantener y proveer informacion con acceso controlado

6. Ciclo de Vida SDLC y Desarrollo de BD

System Development Life Cycle

El desarrollo de bases de datos sigue las fases del SDLC, mapeando cada etapa a actividades especificas de BD:

Fase SDLC Actividad de BD
Identificacion y Planeamiento Modelamiento de la empresa, identificacion del negocio
Analisis Modelamiento conceptual de datos (DER)
Diseno Logico Diseno logico de la base de datos (modelo relacional)
Diseno Fisico Diseno y definicion fisica de la base de datos
Implementacion Implementacion de la base de datos (DDL, carga de datos)
Mantenimiento Mantenimiento de la base de datos

8. Terminologia Clave

  • Transaccion: Operacion completa de lectura o escritura sobre la base de datos.
  • Database Administrator (DBA): Administrador de base de datos responsable de la gestion centralizada.
  • Metadata: Describe las propiedades y caracteristicas de los datos y de su contexto.
