Insight del Arquitecto

Los JOINs son el corazon del modelo relacional. Conocer cuando usar cada tipo (y como el optimizador los ejecuta) marca la diferencia entre consultas que responden en milisegundos y las que colapsan el servidor. Las subconsultas son elegantes pero no siempre eficientes; en PostgreSQL moderno, muchas subconsultas pueden reescribirse como JOINs o con LATERAL para mejor rendimiento.

1. Concepto de JOIN

Un JOIN se emplea para consultar datos que se encuentran en más de una tabla. Las filas son enlazadas empleando valores comunes, típicamente entre la clave primaria (PK) y la clave foránea (FK). Sin una condición de JOIN válida, se genera un producto cartesiano (todas las combinaciones posibles).

Cross Join (Producto Cartesiano):

Se genera cuando la condición de JOIN se omite o es inválida. Todas las filas de la primera tabla se juntan con todas las de la segunda tabla.

-- Consulta de Producto Cartesiano
SELECT EMP.empno, EMP.ename, DEPT.deptno, DEPT.dname
FROM   emp CROSS JOIN dept;
-- O equivalentemente: SELECT * FROM emp, dept;

Si EMP tiene 14 filas y DEPT tiene 4 filas, el resultado tendrá 56 filas (14 × 4):

2. Variables Tupla (Alias)

Simplifican las consultas mediante alias en los nombres de tablas. Son obligatorias cuando hay nombres de columna identicos entre tablas.

SELECT C.nombre, OP.numero_orden, OP.numero_embarques
FROM   orden_pedido OP JOIN cliente C
  ON   OP.id_cliente = C.id_cliente;

3. Tipos de JOIN

Equijoin

Resulta cuando el valor de una columna corresponde explicitamente al valor de alguna columna de otra tabla, usando el operador =.

SELECT table1.column, table2.column
FROM   table1 JOIN table2
  ON   table1.column1 = table2.column2;

-- Ejemplo practico
SELECT C.nombre, OP.numero_orden, OP.numero_embarques
FROM   orden_pedido OP JOIN cliente C
  ON   OP.id_cliente = C.id_cliente;

Cada nombre de columna debe ser precedido por el nombre de la tabla (o su alias). Si las columnas tienen el mismo nombre en ambas tablas, el alias es obligatorio.

Non-Equijoin

Resulta cuando la condicion de JOIN utiliza un operador diferente a la igualdad: BETWEEN, >, <, >=, <=, <>.

SELECT L.nombre_linea
FROM   producto P JOIN linea_producto L
  ON   P.precio BETWEEN L.meta_venta_min AND L.meta_venta_max
  AND  P.idprod = L.idprod;

Self Join

Utilizado para unir filas de una misma tabla con relacion recursiva. Se simulan dos tablas en el FROM creando 2 alias diferentes para la misma tabla.

SELECT E.nombre || 'trabaja para ' || J.nombre
FROM   empleado E JOIN empleado J
  ON   E.cod_jefe = J.cod_emp;

-- Explicacion: E es el empleado, J es su jefe.
-- E.cod_jefe = J.cod_emp relaciona cada empleado con su jefe.

Precaucion: En Self Join siempre use alias obligatorios. Sin alias, el SQL no puede distinguir entre las dos instancias de la misma tabla.

Outer Join

Se utiliza para ver filas que normalmente no se presentan mediante una condicion INNER JOIN. El operador puede ser LEFT o RIGHT, segun el lado del JOIN que no tenga un valor coincidente.

Sintaxis basica

SELECT table1.column, table2.column
FROM   table1 LEFT OUTER JOIN table2
  ON   table1.column1 = table2.column2;
-- Incluye todas las filas de table1, aunque no tengan correspondencia en table2

Ejemplo LEFT JOIN: Todos los clientes, incluso aquellos sin ordenes de pedido.

SELECT nombre, numero_orden, numero_embarques
FROM   cliente LEFT OUTER JOIN orden_pedido
  ON   cliente.id_cliente = orden_pedido.id_cliente;

Ejemplo RIGHT JOIN: Equivalente al anterior, intercambiando el orden de las tablas.

SELECT nombre, numero_orden, numero_embarques
FROM   orden_pedido RIGHT OUTER JOIN cliente
  ON   cliente.id_cliente = orden_pedido.id_cliente;
Tipo de JOIN Descripcion Filas sin match
INNER JOIN Solo filas que coinciden en ambas tablas Excluidas
LEFT OUTER JOIN Todas las filas de la tabla izquierda Tabla derecha = NULL
RIGHT OUTER JOIN Todas las filas de la tabla derecha Tabla izquierda = NULL
FULL OUTER JOIN Todas las filas de ambas tablas Lado sin match = NULL

4. Subconsultas

Una subconsulta es una consulta anidada dentro de otra consulta (principal o externa). Se ejecutan primero y su resultado es usado por la consulta principal a traves de las clausulas WHERE o HAVING.

Sintaxis general

SELECT select_list
FROM   table
WHERE  expr operator (
  SELECT select_list
  FROM   table
);

Reglas para Subconsultas

  • Se usan cuando una consulta esta basada en valores desconocidos.
  • Deben estar encerradas entre parentesis ().
  • Dos clases de operadores: de fila simple (= > >= < <= <>) y de fila multiple (IN, NOT IN).
  • El operador debe aparecer al lado izquierdo de la subconsulta.
  • No pueden contener una clausula ORDER BY (a menos que usen OFFSET/FETCH en versiones modernas).
  • Las subconsultas son procesadas primero; luego la consulta principal usa el resultado.

Procesamiento de Subconsultas Anidadas

Ejemplo paso a paso:

SELECT numero_embarques
FROM   orden_pedido
WHERE  id_cliente = (
  SELECT id_cliente
  FROM   cliente
  WHERE  nombre = 'JUAN'
);

-- 1. La subconsulta se ejecuta primero:
--    SELECT id_cliente FROM cliente WHERE nombre = 'JUAN'  → retorna 990
-- 2. El resultado (990) se pasa a la consulta principal:
--    SELECT numero_embarques FROM orden_pedido WHERE id_cliente = 990

Subconsultas de Fila Simple

La subconsulta retorna una sola fila. Solo usa operadores de fila simple: = > >= < <= <>.

-- Cliente que ha generado el maximo numero de embarques
SELECT id_cliente
FROM   orden_pedido
WHERE  numero_embarques = (
  SELECT MAX(numero_embarques)
  FROM   orden_pedido
);

Si la subconsulta retorna mas de una fila y se usa un operador de fila simple, el motor lanza un error: ORA-01427: single-row subquery returns more than one row. En PostgreSQL: ERROR: more than one row returned by a subquery used as an expression.

Subconsultas de Fila Multiple

La subconsulta retorna varias filas de resultado. Se usan operadores de conjunto:

Operador Significado
IN Igual a cualquier miembro de la lista
NOT IN No igual a ningun miembro de la lista 
-- Clientes NACIONALES o INTERNACIONALES con ordenes de pedido
SELECT numero_embarques
FROM   orden_pedido
WHERE  id_cliente IN (
  SELECT id_cliente
  FROM   cliente
  WHERE  tipo = 'NAC' OR tipo = 'INT'
);

Insight del Arquitecto

En PostgreSQL, las subconsultas en WHERE se ejecutan una vez (subconsultas no correlacionadas) y se cachean. Para subconsultas correlacionadas, considera reescribirlas como JOINs con LATERAL o usar funciones ventana (ROW_NUMBER()) para mejor rendimiento en conjuntos grandes. Siempre verifica el plan con EXPLAIN ANALYZE.
