PostgreSQL Internals: DML y MVCC

PostgreSQL implementa control de concurrencia mediante MVCC (Multi-Version Concurrency Control). Cuando ejecuta un UPDATE o DELETE, el motor no sobrescribe ni borra físicamente la fila en el disco de inmediato:

  • DELETE: Marca la tupla existente como "muerta" (inactiva) al registrar el ID de transacción de eliminación.
  • UPDATE: Funciona como un DELETE de la tupla antigua y un INSERT de la nueva tupla con los datos actualizados.
  • Bloating: El espacio ocupado por tuplas muertas no se libera automáticamente al sistema operativo. Se requiere el proceso de VACUUM para limpiar estas tuplas y hacer reutilizable el espacio en las páginas de datos de 8KB.

Lectura del Plan de Ejecución con EXPLAIN

Para auditar el rendimiento de una consulta DML (como un SELECT de filtrado), ejecute EXPLAIN (ANALYZE, BUFFERS). Esto forzará la ejecución de la consulta de forma segura y retornará métricas reales de:

  • Sec Scan (Sequential Scan): Lectura de todas las páginas de la tabla. Costoso para tablas grandes.
  • Shared Read (I/O de disco): Páginas leídas directamente desde el sistema de almacenamiento cloud.
  • Shared Hit (Buffer Cache): Páginas recuperadas instantáneamente desde la memoria RAM del servidor de BD.

Insight del Arquitecto: Las sentencias DML son las mas usadas en cualquier aplicacion. En PostgreSQL, recursos como RETURNING e INSERT ... ON CONFLICT (upsert) ahorran viajes de ida y vuelta al servidor. Siempre prefiera TRUNCATE sobre DELETE sin WHERE para vaciar tablas grandes.

1. Lenguajes de la Base de Datos Relacional

Una base de datos relacional se maneja a traves de distintos sublenguajes SQL:

  • DML (Data Manipulation Language) - INSERT, UPDATE, DELETE, SELECT
  • DDL (Data Definition Language) - CREATE, ALTER, DROP, RENAME
  • DCL (Data Control Language) - GRANT, REVOKE
  • TCL (Transaction Control) - COMMIT, ROLLBACK, SAVEPOINT

2. INSERT - Insercion de Datos

Aniade una nueva fila a una tabla existente. Todos los campos con restriccion NOT NULL deben recibir un valor.

INSERT INTO table [column, column...] VALUES (valor1, valor2, ...);

Ejemplo: Tabla cliente y orden_pedido.

CREATE TABLE cliente ( codigo_cliente NUMBER(5) PRIMARY KEY, nombre VARCHAR2(25) NOT NULL, apellido VARCHAR2(50) NOT NULL, telefono VARCHAR2(10) NOT NULL ); CREATE TABLE orden_pedido ( numero_orden NUMBER(7) PRIMARY KEY, fecha_colocacion DATE, fecha_llenado DATE, numero_embarques NUMBER(5), id_cliente NUMBER(7) REFERENCES cliente ); INSERT INTO cliente VALUES (1, 'JOSE', 'PARDO', '01-4414896'); INSERT INTO orden_pedido VALUES (555, '25-OCT-07', '05-OCT-07', 5, 1);

Nota: Se puede especificar la lista de columnas. El valor NULL es valido para campos sin restriccion NOT NULL.

3. UPDATE - Modificacion de Datos

Modifica el valor de uno o varios campos en filas existentes. Puede elegir las filas mediante la clausula WHERE.

UPDATE table SET column = nuevo_valor | expresion [WHERE condicion {AND|OR} condicion...];

Ejemplo: Cambiar numero de embarques de la orden 555 y telefono del cliente 1.

UPDATE orden_pedido SET numero_embarques = 3 WHERE numero_orden = 555; UPDATE cliente SET telefono = '01-4367788' WHERE codigo_cliente = 1;

Precaucion: Si omite la clausula WHERE, se actualizan TODAS las filas de la tabla. Siempre verifique primero con un SELECT.

4. DELETE - Eliminacion de Datos

Elimina una o mas filas de una tabla. La clausula WHERE determina cuales filas se eliminan.

DELETE FROM table [WHERE condicion {AND|OR} condicion...];

Ejemplo: Eliminar la orden de pedido numero 555.

DELETE FROM orden_pedido WHERE numero_orden = 555;

5. SELECT - Consulta de Datos

Permite consultar la informacion contenida en una o varias tablas. Es la instruccion DML mas utilizada.

SELECT ALL|*|DISTINCT|(column, column...) FROM table [WHERE condicion {AND|OR} condicion...];
  • ALL o * muestra todas las columnas y todas las filas
  • DISTINCT elimina duplicados en los resultados
  • WHERE filtra las filas segun una condicion logica

Ejemplos de SELECT:

SELECT ALL FROM orden_pedido; SELECT * FROM cliente; SELECT * FROM cliente WHERE codigo_cliente = 1; SELECT DISTINCT nombre FROM cliente; SELECT nombre, apellido FROM cliente WHERE codigo_cliente = 1;

Insight del Arquitecto: En PostgreSQL, la paginacion se logra con LIMIT ... OFFSET ... a diferencia de Oracle que usa ROWNUM o SQL Server con TOP. La clausula RETURNING es exclusiva de PostgreSQL y permite devolver los valores generados (como SERIAL) tras una insercion.

💡 Tips del Profesor

•

Evite SELECT * en producción: Recuperar columnas innecesarias incrementa el tráfico de red, sobrecarga la memoria RAM y bloquea la capacidad del planificador de usar Index-Only Scans.

•

Autovacuum Tuning: Asegúrese de configurar correctamente los parámetros de autovacuum en su instancia de RDS para evitar el temido 'Table Bloating' en tablas con alto volumen de modificaciones.

•

Cuidado con DELETE sin WHERE: Borrar millones de registros uno a uno genera millones de registros en el WAL (Write-Ahead Log) y genera millones de tuplas muertas. Si desea vaciar una tabla completa, prefiera TRUNCATE TABLE.

✍️ Autoevaluación y Ejercicios Prácticos

1 ¿Qué ocurre internamente en PostgreSQL cuando se ejecuta un UPDATE de una fila?

Se marca la tupla actual como inactiva (muerta) y se inserta una nueva tupla (versión física) en una página de la base de datos con los nuevos valores y el ID de transacción de inserción.

2 ¿Por qué es importante utilizar EXPLAIN (ANALYZE, BUFFERS) en lugar de un simple EXPLAIN?

Porque EXPLAIN solo calcula una estimación teórica del planificador, mientras que ANALYZE y BUFFERS ejecuta la consulta real mostrando tiempos exactos e impacto de I/O en caché y disco.

3 ¿Qué ventaja tiene TRUNCATE sobre un DELETE sin cláusula WHERE?

TRUNCATE es una instrucción DDL que libera instantáneamente el espacio en disco eliminando los archivos de datos subyacentes y requiere muy pocos recursos de WAL, a diferencia de DELETE que procesa fila por fila.

  • DELETE Eliminar filas de una tabla
