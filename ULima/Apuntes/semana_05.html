1. Lenguajes de la Base de Datos Relacional

Una base de datos relacional se maneja a traves de distintos sublenguajes SQL:

  • DML (Data Manipulation Language) - INSERT, UPDATE, DELETE, SELECT
  • DDL (Data Definition Language) - CREATE, ALTER, DROP, RENAME
  • DCL (Data Control Language) - GRANT, REVOKE
  • TCL (Transaction Control) - COMMIT, ROLLBACK, SAVEPOINT

2. INSERT - Insercion de Datos

Aniade una nueva fila a una tabla existente. Todos los campos con restriccion NOT NULL deben recibir un valor.

INSERT INTO table [column, column...]
VALUES (valor1, valor2, ...);

Ejemplo: Tabla cliente y orden_pedido.

CREATE TABLE cliente (
    codigo_cliente NUMBER(5) PRIMARY KEY,
    nombre VARCHAR2(25) NOT NULL,
    apellido VARCHAR2(50) NOT NULL,
    telefono VARCHAR2(10) NOT NULL
);

CREATE TABLE orden_pedido (
    numero_orden NUMBER(7) PRIMARY KEY,
    fecha_colocacion DATE,
    fecha_llenado DATE,
    numero_embarques NUMBER(5),
    id_cliente NUMBER(7) REFERENCES cliente
);

INSERT INTO cliente
VALUES (1, 'JOSE', 'PARDO', '01-4414896');

INSERT INTO orden_pedido
VALUES (555, '25-OCT-07', '05-OCT-07', 5, 1);

Nota: Se puede especificar la lista de columnas. El valor NULL es valido para campos sin restriccion NOT NULL.

3. UPDATE - Modificacion de Datos

Modifica el valor de uno o varios campos en filas existentes. Puede elegir las filas mediante la clausula WHERE.

UPDATE table
SET column = nuevo_valor | expresion
[WHERE condicion {AND|OR} condicion...];

Ejemplo: Cambiar numero de embarques de la orden 555 y telefono del cliente 1.

UPDATE orden_pedido
SET numero_embarques = 3
WHERE numero_orden = 555;

UPDATE cliente
SET telefono = '01-4367788'
WHERE codigo_cliente = 1;

Precaucion: Si omite la clausula WHERE, se actualizan TODAS las filas de la tabla. Siempre verifique primero con un SELECT.

4. DELETE - Eliminacion de Datos

Elimina una o mas filas de una tabla. La clausula WHERE determina cuales filas se eliminan.

DELETE FROM table
[WHERE condicion {AND|OR} condicion...];

Ejemplo: Eliminar la orden de pedido numero 555.

DELETE FROM orden_pedido
WHERE numero_orden = 555;

5. SELECT - Consulta de Datos

Permite consultar la informacion contenida en una o varias tablas. Es la instruccion DML mas utilizada.

SELECT ALL|*|DISTINCT|(column, column...)
FROM table
[WHERE condicion {AND|OR} condicion...];
  • ALL o * muestra todas las columnas y todas las filas
  • DISTINCT elimina duplicados en los resultados
  • WHERE filtra las filas segun una condicion logica

Ejemplos de SELECT:

SELECT ALL FROM orden_pedido;
SELECT * FROM cliente;
SELECT * FROM cliente WHERE codigo_cliente = 1;
SELECT DISTINCT nombre FROM cliente;
SELECT nombre, apellido FROM cliente
WHERE codigo_cliente = 1;

Insight del Arquitecto: En PostgreSQL, la paginacion se logra con LIMIT ... OFFSET ... a diferencia de Oracle que usa ROWNUM o SQL Server con TOP. La clausula RETURNING es exclusiva de PostgreSQL y permite devolver los valores generados (como SERIAL) tras una insercion.
