La Arquitectura del Lenguaje Procedimental PL/pgSQL

PL/pgSQL es un lenguaje de programación procedural estructurado que amplía las capacidades declarativas de SQL. Permite la ejecución de lógica compleja directamente en el servidor de base de datos a través de:

  • Bloques Estructurados: Uso de secciones DECLARE, BEGIN y END;.
  • Tipado Seguro: Uso de atributos dinámicos como %TYPE (hereda el tipo de una columna) y %ROWTYPE (hereda la estructura completa de una fila).
  • Control de Flujo: Estructuras de decisión (IF-THEN-ELSE) y bucles iterativos avanzados para recorrer cursores.

Optimización de PL/pgSQL en Instancias Gestionadas

Una ventaja crítica de PL/pgSQL es que el motor compila y cachea los planes de ejecución de las sentencias SQL contenidas en sus funciones la primera vez que se invocan dentro de una sesión de base de datos.

Gotcha de Memoria: El uso excesivo de variables temporales de tipo registro o el retorno de grandes conjuntos de datos usando cursores explícitos sin límites puede saturar la memoria RAM asignada a la sesión del cliente en RDS.

Objetivos de Aprendizaje

  • Comprender la estructura de un bloque PL/SQL: DECLARE, BEGIN, EXCEPTION, END.
  • Declarar variables usando tipado directo, %TYPE y %ROWTYPE.
  • Usar estructuras de control: IF, LOOP, WHILE, FOR.
  • Implementar cursores implicitos y explicitos para procesamiento fila a fila.

Insight del Arquitecto

PL/SQL (y su equivalente en PostgreSQL, PL/pgSQL) permite mover la logica de negocio al servidor de base de datos. Esto reduce el trafico de red y centraliza las reglas. Sin embargo, use este poder con responsabilidad: la logica compleja en la base de datos es mas dificil de versionar, probar y escalar horizontalmente. Reserve PL/SQL para operaciones intensivas en datos (ETL, validacion de integridad, auditoria).

1. Estructura de un Programa PL/SQL

Todo programa PL/SQL se compone de cuatro secciones, aunque solo BEGIN y END son obligatorias:

Seccion Obligatorio Descripcion
DECLARE Opcional Variables, restricciones, cursores y excepciones definidas por el usuario
BEGIN Obligatorio Sentencias SQL y sentencias de control PL/SQL
EXCEPTION Opcional Acciones a ejecutar cuando ocurren errores
END; Obligatorio Cierra el bloque

Ejemplo: Bloque anonimo basico

DECLARE v_mensaje VARCHAR2(50) := 'Hola, PL/SQL!'; BEGIN DBMS_OUTPUT.PUT_LINE(v_mensaje); END; /

2. Declaracion de Variables

Las variables se declaran en la seccion DECLARE con la siguiente sintaxis:

identificador [CONSTANT] tipo_dato [NOT NULL] [:= | DEFAULT expr];

Ejemplos de declaracion

DECLARE v_estado_civil CHAR(1); v_total_sal NUMBER(9,2) := 0; v_fecha_orden DATE := SYSDATE + 7; v_valido BOOLEAN NOT NULL := TRUE; v_cod_cuenta VARCHAR2(5) NOT NULL := 'AP001'; v_credito_limite CONSTANT NUMBER := 5000.00; v_estado BOOLEAN DEFAULT FALSE; BEGIN NULL; -- sentencia que no hace nada (placeholder) END; /

Tipos Especiales: %TYPE y %ROWTYPE

PL/SQL ofrece tipos derivados que se adaptan automaticamente a la estructura de la base de datos:

%TYPE Hereda el tipo de dato de una columna existente.

v_debito cuenta.credito%TYPE; -- mismo tipo que cuenta.credito

%ROWTYPE Hereda la estructura completa de una tabla (todas sus columnas).

v_dept_reg dept%ROWTYPE; -- v_dept_reg.deptno, v_dept_reg.dname, v_dept_reg.loc disponibles

Registros definidos por el usuario (RECORD)

TYPE emp_record_type IS RECORD ( apellido VARCHAR2(25), cargo_id VARCHAR2(10), salario NUMBER(8,2) );

Tablas PL/SQL (arrays asociativos)

TYPE dept_table_type IS TABLE OF emp.apellido%TYPE INDEX BY BINARY_INTEGER;

3. Funciones Built-in (Manipulación y Conversión)

PL/SQL incluye funciones integradas para la manipulación de caracteres, cálculos numéricos, operaciones de fecha y conversiones de tipo.

Funciones de Manipulación de Caracteres

Función Descripción / Ejemplo Resultado
CONCAT(str1, str2) Concatena dos cadenas: CONCAT('Hola', 'Mundo') "HolaMundo"
SUBSTR(str, pos, len) Extrae parte de una cadena: SUBSTR('HolaMundo', 1, 5) "HolaM"
LENGTH(str) Retorna la longitud de la cadena: LENGTH('HolaMundo') 9
INSTR(str, char) Encuentra la posición de un carácter: INSTR('HolaMundo', 'M') 5
LPAD(col, len, pad) Rellena a la izquierda: LPAD(salario, 10, '*') "*****24000"
RPAD(col, len, pad) Rellena a la derecha: RPAD(salario, 10, '*') "24000*****"
TRIM(char FROM str) Remueve caracteres extremos: TRIM('H' FROM 'HolaMundo') "olaMundo"

Funciones Numéricas

Función Descripción / Ejemplo Resultado
ROUND(n, d) Redondea a d decimales: ROUND(45.926, 2) 45.93
TRUNC(n, d) Trunca a d decimales: TRUNC(45.926, 2) 45.92
MOD(m, n) Retorna el residuo de la división: MOD(1600, 300) 100

Funciones de Fecha

Función Descripción / Ejemplo Resultado
MONTHS_BETWEEN(d1, d2) Meses entre d1 y d2: MONTHS_BETWEEN('01-SEP-95', '11-JAN-94') 19.6774194
ADD_MONTHS(d, n) Suma n meses a una fecha: ADD_MONTHS('11-JAN-94', 6) 11-JUL-94
NEXT_DAY(d, day) Próximo día especificado: NEXT_DAY('10-JUN-08', 'TUESDAY') 17-JUN-08
LAST_DAY(d) Último día del mes: LAST_DAY('01-FEB-08') 29-FEB-08
ROUND(d, 'YEAR'|'MONTH') Redondea la fecha al año o mes más cercano Fecha redondeada
TRUNC(d, 'YEAR'|'MONTH') Trunca la fecha al inicio de año o mes Fecha truncada

Funciones de Conversión de Datos

Permiten convertir tipos de datos de forma explícita:

  • TO_CHAR(n|d, format): Convierte número o fecha a cadena de caracteres. Ejemplos:
    TO_CHAR(55000, '$99,999.00') -- Retorna "$55,000.00" TO_CHAR(TRUNC(SYSDATE), 'DD MONTH YYYY') -- Retorna p.ej. "26 JUNIO 2026"
  • TO_DATE(str, format): Convierte cadena a tipo DATE. Ejemplos:
    TO_DATE('25/12/2007', 'DD/MM/YYYY') TO_DATE('01-Jan-90', 'DD-Mon-RR')
  • TO_NUMBER(str): Convierte cadena a tipo numérico. Ejemplos:
    TO_NUMBER('0100') -- Retorna 100 TO_NUMBER('1000.10') -- Retorna 1000.10

Otras Funciones del Sistema

  • NVL(expr1, expr2): Sustituye valores nulos. Retorna expr2 si expr1 es NULL.
  • NVL2(expr1, expr2, expr3): Si expr1 no es NULL retorna expr2, de lo contrario retorna expr3.
  • DECODE(col|expr, busqueda1, res1, busqueda2, res2, ..., default): Estructura condicional inline (similar a un IF-THEN-ELSE o CASE en SQL).

4. Estructuras de Control

Condicional: IF

IF condicion1 THEN secuencia_de_instrucciones1; ELSIF condicion2 THEN secuencia_de_instrucciones2; ELSE secuencia_de_instrucciones3; END IF;

Bucles

LOOP basico con EXIT WHEN

LOOP secuencia_de_instrucciones; EXIT WHEN condicion; END LOOP;

WHILE LOOP

WHILE condicion LOOP secuencia_de_instrucciones; END LOOP;

FOR LOOP (con y sin REVERSE)

FOR contador IN 1..10 LOOP secuencia_de_instrucciones; END LOOP; FOR contador IN REVERSE 10..1 LOOP secuencia_de_instrucciones; END LOOP;

5. Cursores

Un cursor es un puntero a un conjunto de resultados. Permite procesar fila por fila cuando se necesita logica procedural que no puede expresarse en SQL puro.

Cursores Implicitos

Toda sentencia SQL en PL/SQL tiene un cursor implicito asociado. Sus atributos permiten monitorear la operacion:

Atributo Tipo Descripcion
%ISOPEN Booleano TRUE si el cursor esta abierto
%NOTFOUND Booleano TRUE si el ultimo FETCH no trajo filas
%FOUND Booleano TRUE mientras el ultimo FETCH trae una fila
%ROWCOUNT Numerico Total de filas retornadas hasta el momento

Cursores Explicitos: Ciclo de Vida

El ciclo completo de un cursor explicito tiene 4 pasos:

Paso Comando Descripcion
1 DECLARE CURSOR Crear el cursor con la sentencia SELECT
2 OPEN Abrir el cursor (ejecuta el SELECT)
3 FETCH Cargar la fila actual en variables
4 CLOSE Cerrar el cursor

Programa Simple con Cursor (FETCH explicito)

DECLARE CURSOR ejecursor IS SELECT nombre, salario FROM empleado WHERE salario < 5000; v_nombre empleado.nombre%TYPE; v_salario empleado.salario%TYPE; BEGIN OPEN ejecursor; LOOP FETCH ejecursor INTO v_nombre, v_salario; EXIT WHEN ejecursor%NOTFOUND; INSERT INTO TEMP VALUES (v_nombre, v_salario); END LOOP; CLOSE ejecursor; END; /

Programa Simple con Cursor (FOR LOOP simplificado)

El FOR LOOP simplifica el manejo: abre, fetchea y cierra automaticamente el cursor.

DECLARE CURSOR ejecursor IS SELECT nombre, salario FROM empleado WHERE salario < 5000; BEGIN FOR ejecursor_rec IN ejecursor LOOP INSERT INTO TEMP VALUES (ejecursor_rec.nombre, ejecursor_rec.salario); END LOOP; END; /

Diferencia clave: Con FETCH explicito controlamos manualmente apertura, lectura y cierre. Con FOR LOOP, PL/SQL maneja automaticamente el ciclo de vida del cursor. El FOR LOOP es mas seguro (no hay riesgo de olvidar CLOSE) y mas legible.

Excepciones Predefinidas

Excepcion Causa
NO_DATA_FOUND SELECT INTO no retorna filas
TOO_MANY_ROWS SELECT INTO retorna mas de una fila
DUP_VAL_ON_INDEX Insercion de valor duplicado en columna unica
ZERO_DIVIDE Division entre cero
INVALID_CURSOR Operacion ilegal sobre un cursor
CURSOR_ALREADY_OPEN Intento de abrir un cursor ya abierto
VALUE_ERROR Error de conversion o truncamiento

Insight del Arquitecto

En PostgreSQL, PL/pgSQL ofrece el mismo paradigma con sintaxis ligeramente diferente: DECLARE usa $function$ o LANGUAGE plpgsql, los cursores se declaran con CURSOR FOR y el FOR LOOP con FOR registro IN consulta LOOP. La principal ventaja sobre Oracle PL/SQL es que PostgreSQL permite RETURN QUERY para devolver conjuntos directamente desde funciones. Siempre prefiera operaciones basadas en conjuntos (SET-based) sobre cursores: un solo UPDATE con WHERE suele ser mas rapido que un cursor con 10,000 FETCHs.

💡 Tips del Profesor

•

Utilice %TYPE y %ROWTYPE: Evite harcodear tipos de datos en la declaración de variables de PL/pgSQL; si la columna física cambia en el DDL futuro, el bloque procedimental se adaptará automáticamente sin romperse.

•

Evite lógica procedimental cuando baste SQL nativo: Las sentencias SQL declarativas son optimizadas de forma mucho más eficiente por el planificador que los bucles LOOP procedimentales fila por fila.

•

Pruebe bloques anónimos con ROLLBACK: Para validar la lógica de scripts PL/pgSQL en producción, encapsule el bloque con un BEGIN; ... ROLLBACK; para evitar cambios permanentes accidentales.

✍️ Autoevaluación y Ejercicios Prácticos

1 ¿Qué ventaja tiene el uso de %TYPE en la declaración de variables?

Garantiza la consistencia del tipo de datos con la columna física subyacente de la tabla, de modo que si se modifica el tipo de columna en el futuro mediante DDL, la función PL/pgSQL se mantiene válida automáticamente sin refactorizaciones.

2 ¿Cómo maneja PostgreSQL el plan de ejecución de las consultas dentro de un bloque PL/pgSQL?

El motor compila la consulta y guarda el plan de ejecución en la memoria caché de la conexión actual. En invocaciones subsiguientes se reutiliza el plan, eliminando el costo de parsing y planificación.

3 ¿Qué diferencia hay entre un bloque anónimo DO y una función almacenada?

Un bloque anónimo DO se compila y ejecuta al vuelo en una sola llamada y no se almacena de forma persistente en el diccionario de datos ni acepta parámetros de entrada, a diferencia de las funciones.

Conceptos Clave

  • § Bloque PL/SQLUnidad basica: DECLARE + BEGIN + EXCEPTION + END.
  • § %TYPEDeclara variable con el tipo de una columna existente.
  • § %ROWTYPEDeclara registro con la estructura completa de una tabla.
  • § CursorPuntero a un conjunto de resultados para recorrer fila por fila.
  • § FETCHInstruccion que carga la fila actual del cursor en variables.
