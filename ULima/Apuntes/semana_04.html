?? Insight del Arquitecto: DML en PostgreSQL

Diferencias con Oracle/SQL Server:

  • LIMIT vs TOP (SQL Server) o ROWNUM (Oracle)
  • LIMIT 10 OFFSET 20 para paginación
  • RETURNING * para obtener datos insertados
  • INSERT ... ON CONFLICT (upsert)
  • TRUNCATE vs DELETE (más rápido para vaciar tablas)

1. Inserción de Datos (INSERT)

Sintaxis básica:

INSERT INTO tabla (columna1, columna2, ...)
VALUES (valor1, valor2, ...);

Ejemplos:

-- Insertar todos los campos (en orden)
INSERT INTO cliente 
VALUES (1, 'JUAN', 'PEREZ', '01-4414896');

-- Insertar campos específicos
INSERT INTO cliente (nombre, apellido, email) 
VALUES ('MARIA', 'GARCIA', 'maria@email.com');

-- Insertar múltiples filas
INSERT INTO producto (nombre, precio) VALUES
    ('Laptop', 1500.00),
    ('Mouse', 25.50),
    ('Teclado', 45.00);

-- Insertar con RETURNING (PostgreSQL)
INSERT INTO cliente (nombre, apellido) 
VALUES ('PEDRO', 'LOPEZ')
RETURNING id_cliente, nombre;

Regla: Los campos con restricción NOT NULL deben ingresarse. Los demás pueden ser NULL o usar DEFAULT.

2. Actualización de Datos (UPDATE)

Sintaxis:

UPDATE tabla
SET columna1 = nuevo_valor, columna2 = nuevo_valor
WHERE condición;

Ejemplos:

-- Actualizar un campo específico
UPDATE cliente
SET telefono = '01-4367788'
WHERE id_cliente = 1;

-- Actualizar múltiples campos
UPDATE producto
SET precio = precio * 1.10, stock = stock - 5
WHERE id_producto = 10;

-- Actualizar con cálculo
UPDATE orden_pedido
SET numero_embarques = numero_embarques + 1
WHERE estado = 'pendiente';

?? Peligro: Si omites el WHERE, se actualizan TODAS las filas. Siempre prueba con SELECT primero:

SELECT * FROM tabla WHERE condición;  -- verificar primero
UPDATE tabla SET ... WHERE condición;  -- luego actualizar

3. Eliminación de Datos (DELETE)

Sintaxis:

DELETE FROM tabla
WHERE condición;

Ejemplos:

-- Eliminar un registro específico
DELETE FROM orden_pedido
WHERE numero_orden = 555;

-- Eliminar varios registros
DELETE FROM cliente
WHERE tipo = 'INACTIVO' AND fecha_registro < '2020-01-01';

-- Eliminar todos los registros (ˇcuidado!)
DELETE FROM tabla_temp;  -- elimina todas las filas

-- Usar TRUNCATE (más eficiente en PostgreSQL)
TRUNCATE TABLE tabla_temp RESTART IDENTITY;

4. Consulta de Datos (SELECT)

Formas básicas:

-- Seleccionar todas las columnas
SELECT * FROM cliente;

-- Seleccionar columnas específicas
SELECT nombre, apellido, email FROM cliente;

-- Usar alias
SELECT nombre AS "Nombre del Cliente", 
       precio AS "Precio Unitario"
FROM producto;

-- DISTINCT: valores únicos
SELECT DISTINCT tipo FROM cliente;

-- ORDER BY: ordenar resultados
SELECT * FROM producto
ORDER BY precio DESC;  -- DESC = descendente

SELECT * FROM cliente
ORDER BY apellido ASC, nombre ASC;  -- múltiples criterios

5. Filtrado con WHERE

OperadorDescripción
=Igual a
<> o !=Diferente de
> <Mayor/Menor que
>= <=Mayor/Menor o igual
BETWEENEntre rango (inclusive)
LIKEPatrón de texto
INEstá en lista
IS NULL / IS NOT NULLVerificar nulos

Ejemplos de WHERE:

-- Filtrar por igualdad
SELECT * FROM producto WHERE precio = 100;

-- Filtrar por rango
SELECT * FROM producto 
WHERE precio BETWEEN 50 AND 200;

-- Patrones con LIKE
SELECT * FROM cliente 
WHERE nombre LIKE 'J%';  -- comienza con J

SELECT * FROM producto 
WHERE nombre LIKE '%lap%';  -- contiene "lap"

-- Lista con IN
SELECT * FROM cliente 
WHERE tipo IN ('PREMIUM', 'VIP');

-- Verificar nulos
SELECT * FROM cliente 
WHERE email IS NOT NULL;

-- Combinaciones AND/OR
SELECT * FROM producto
WHERE precio > 100 AND stock > 0;

Operador LIKE: % = cualquier??, _ = un caracter. Ej: 'J__n' = nombres de 4 letras starting con J.
