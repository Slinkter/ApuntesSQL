?? Insight del Arquitecto: El Futuro de las BD

El panorama en 2026: Ya no hay una solución única. El enfoque políglota de persistencia (polyglot persistence) es la norma. Las organizaciones usan múltiples tipos de BD según el caso de uso: PostgreSQL/MySQL para transaccional, MongoDB/Cassandra para documentos y datos masivos, Redis para caché, Elastic para búsqueda.

2. Particionamiento de Tablas

Partitioning: Dividir una tabla grande en partes más pequeñas (particiones) que se pueden gestionar independientemente.

Tipos de Particionamiento:

  • Por Rango (Range): Por intervalos de valores (ej: por fecha)
  • Por Lista (List): Por valores específicos (ej: por región)
  • Por Hash (Hash): Distribución uniforme

Ejemplo en PostgreSQL:



-- Particionar tabla de pedidos por fecha

CREATE TABLE pedido (

    id SERIAL,

    fecha DATE NOT NULL,

    cliente_id INTEGER,

    total NUMERIC

) PARTITION BY RANGE (fecha);



-- Crear particiones

CREATE TABLE pedido_2024 PARTITION OF pedido

    FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');



CREATE TABLE pedido_2025 PARTITION OF pedido

    FOR VALUES FROM ('2025-01-01') TO ('2026-01-01');

4. Seguridad en Bases de Datos

  • Control de Acceso: Roles, permisos GRANT/REVOKE
  • Encriptación: En tránsito (SSL/TLS), en disco (PGP)
  • Row Level Security (RLS): Filtrar filas por usuario
  • Auditoría: Logs de acceso y modificaciones
  • Backup encriptado: Proteger respaldos

RLS en PostgreSQL:



-- Habilitar RLS en tabla

ALTER TABLE pedido ENABLE ROW LEVEL SECURITY;



-- Política: cada cliente ve solo sus pedidos

CREATE POLICY ver_propio_pedido ON pedido

    FOR SELECT

    USING (cliente_id = current_user);

5. Bases de Datos NoSQL

TipoEjemplosUso Ideal
DocumentosMongoDB, CouchDBCatálogos, contenido flexible
Clave-ValorRedis, DynamoDBCache, sesiones
Familia de ColumnasCassandra, HBaseEscritura masiva, time series
GrafosNeo4jRedes, recomendaciones

Características NoSQL:

  • Esquema flexible (schemaless)
  • Escalabilidad horizontal fácil
  • Consistencia eventual (BASE vs ACID)
  • Alta disponibilidad

6. NewSQL yTendencias Modernas

  • NewSQL: Combina ACID de RDBMS con escalabilidad de NoSQL
    Ej: CockroachDB, TiDB, Google Spanner
  • HTAP: Hybrid Transactional/Analytical Processing
    Ej: SingleStore, Apache Druid
  • Vector Databases: Para ML e IA
    Ej: Pinecone, Milvus, pgvector
  • Time Series: IoT, métricas
    Ej: TimescaleDB, InfluxDB

7. PostgreSQL Moderno (v15+)

  • Replicación lógica a nivel de tabla
  • Parallel queries mejoradas
  • JSON path más rápido
  • pg_stat_statements para monitoreo
  • Extension pgvector para embeddings
