# Glosario - Clase 02: Sistema de Gestión de Base de Datos (SGBD)

*   **Sistema de Gestión de Base de Datos (SGBD/DBMS):** Software diseñado para definir, crear, mantener y controlar el acceso a una base de datos, actuando como interfaz entre los usuarios/aplicaciones y los datos.
*   **Lenguaje de Definición de Datos (DDL):** Componente de un lenguaje de base de datos utilizado para crear y modificar la estructura de los objetos de la base de datos (ej. `CREATE`, `ALTER`, `DROP`).
*   **Lenguaje de Manipulación de Datos (DML):** Componente de un lenguaje de base de datos utilizado para insertar, recuperar, modificar y eliminar datos en la base de datos (ej. `SELECT`, `INSERT`, `UPDATE`, `DELETE`).
*   **Control de Acceso (SGBD):** Mecanismos implementados en un SGBD para asegurar que solo los usuarios autorizados puedan acceder a los datos y realizar operaciones permitidas.
*   **Integridad de Datos (SGBD):** La capacidad del SGBD para mantener la precisión, consistencia y fiabilidad de los datos a través de reglas y restricciones definidas.
*   **Recuperación y Respaldo (SGBD):** Funcionalidades de un SGBD para crear copias de seguridad de los datos y restaurar la base de datos a un estado consistente después de un fallo o pérdida de datos.
*   **Control de Concurrencia:** Mecanismo del SGBD que permite a múltiples usuarios acceder y modificar la base de datos simultáneamente sin que sus operaciones interfieran entre sí, garantizando la consistencia.
*   **Catálogo del Sistema (Metadatos):** Un diccionario de datos gestionado por el SGBD que almacena información sobre la estructura de la base de datos, los objetos, los usuarios y los permisos.
*   **Motor de Base de Datos:** El componente central del SGBD que gestiona el almacenamiento, la recuperación y la actualización física de los datos.
*   **Propiedades ACID:** Acrónimo que describe las propiedades fundamentales de las transacciones de base de datos: Atomicidad, Consistencia, Aislamiento y Durabilidad, que garantizan la fiabilidad de las transacciones.
*   **Independencia de Datos:** Principio del SGBD que separa la vista lógica de los datos de su implementación física, permitiendo cambios en un nivel sin afectar al otro.
