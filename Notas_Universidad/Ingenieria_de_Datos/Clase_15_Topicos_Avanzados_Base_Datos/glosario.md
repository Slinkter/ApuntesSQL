# Glosario - Clase 15: Tópicos Avanzados de Bases de Datos

*   **Bases de Datos NoSQL:** Categoría de bases de datos que no siguen el modelo relacional tradicional y están diseñadas para manejar grandes volúmenes de datos distribuidos, esquemas flexibles y alta escalabilidad horizontal.
*   **Bases de Datos Clave-Valor:** Un tipo de base de datos NoSQL que almacena datos como una colección de pares clave-valor, ideal para almacenar objetos simples y con alta velocidad de lectura/escritura.
*   **Bases de Datos Orientadas a Documentos:** Un tipo de base de datos NoSQL que almacena datos en documentos semiestructurados (ej. JSON, BSON), permitiendo esquemas flexibles y cambios ágiles.
*   **Bases de Datos Orientadas a Columnas (Wide-Column Stores):** Un tipo de base de datos NoSQL que almacena datos en familias de columnas, optimizadas para agregaciones y consultas analíticas sobre grandes conjuntos de datos distribuidos.
*   **Bases de Datos Orientadas a Grafos:** Un tipo de base de datos NoSQL que utiliza estructuras de grafos (nodos y aristas) para representar y almacenar datos, ideal para modelar relaciones complejas.
*   **Big Data:** Término que describe volúmenes extremadamente grandes de datos que no pueden ser procesados o analizados con herramientas y métodos tradicionales, caracterizado por sus "3 Vs".
*   **Volumen (Big Data):** Se refiere a la inmensa cantidad de datos generados y almacenados.
*   **Velocidad (Big Data):** Se refiere a la rapidez con la que se generan y se deben procesar los datos (datos en tiempo real).
*   **Variedad (Big Data):** Se refiere a los diversos tipos y formatos de datos (estructurados, semiestructurados, no estructurados) que se manejan.
*   **Hadoop:** Un framework de código abierto para almacenar y procesar grandes conjuntos de datos de forma distribuida en clústeres de computadoras.
*   **Spark:** Un motor de procesamiento de datos rápido y de propósito general para el análisis de Big Data, a menudo utilizado junto con Hadoop.
*   **Bases de Datos en la Nube (Cloud Databases):** Servicios de bases de datos ofrecidos por proveedores de computación en la nube que permiten aprovisionar, escalar y gestionar bases de datos sin la infraestructura local.
*   **DBaaS (Database as a Service):** Un modelo de servicio en la nube donde un proveedor gestiona la base de datos completa para el usuario, incluyendo hardware, software, copias de seguridad y mantenimiento.
*   **Bases de Datos en Memoria (In-Memory Databases - IMDB):** Bases de datos que almacenan los datos principalmente en la memoria RAM del servidor para lograr un rendimiento significativamente más rápido en comparación con las bases de datos basadas en disco.
*   **Procesamiento Paralelo:** Técnica que divide una tarea computacional en subtareas que se ejecutan simultáneamente en múltiples procesadores.
*   **Procesamiento Distribuido:** Técnica que utiliza múltiples computadoras interconectadas (un clúster) para trabajar en una tarea común, distribuyendo los datos y la carga de procesamiento.
*   **Sharding:** Una técnica de particionamiento horizontal de una base de datos, dividiendo grandes tablas en piezas más pequeñas (fragmentos) que se almacenan en diferentes servidores.
*   **Replicación (Bases de Datos):** La creación y mantenimiento de múltiples copias de una base de datos para mejorar la disponibilidad, la tolerancia a fallos y el rendimiento de lectura.
*   **Cifrado de Datos:** Proceso de codificar datos para protegerlos del acceso no autorizado, haciéndolos ilegibles sin la clave de descifrado.
*   **Enmascaramiento de Datos:** Proceso de ocultar datos sensibles con datos ficticios o alterados, manteniendo el formato original, para su uso en entornos que no sean de producción (ej. desarrollo, pruebas).
*   **Auditoría de Bases de Datos:** El proceso de registrar y monitorear las actividades de acceso y modificación de datos en una base de datos para seguridad, cumplimiento y detección de anomalías.
*   **Escalabilidad Horizontal:** La capacidad de un sistema para manejar un aumento en la carga de trabajo añadiendo más servidores o nodos a la infraestructura, común en bases de datos NoSQL.
*   **Latencia:** El tiempo que tarda una operación en completarse, especialmente el retraso entre el envío de una solicitud y la recepción de la respuesta.
